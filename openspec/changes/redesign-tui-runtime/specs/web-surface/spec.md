## ADDED Requirements

### Requirement: Web UI from Shared Components

The system SHALL provide a web-based UI that renders the same React component library used by the terminal TUI. The web surface SHALL be served by a Fastify HTTP server bundled with orc, started via `orc web`.

The web surface SHALL connect to the orc daemon via WebSocket and present the same views, state, and navigation as the TUI — including agent pane rendering via xterm.js (browser canvas mode).

#### Scenario: Web surface mirrors TUI state
- **WHEN** the user runs `orc web` and opens the provided URL in a browser
- **THEN** the web UI shows the same orchestration state as the TUI
- **AND** agent panes render via xterm.js canvas (full terminal fidelity)
- **AND** navigation, command palette, and context menu work with the same keyboard shortcuts
- **AND** state changes in the TUI appear in the web UI in real-time (and vice versa)

#### Scenario: Remote access via web
- **WHEN** the orc daemon is running on machine A
- **AND** a user opens the web UI URL from machine B
- **THEN** the user authenticates with a token (generated by `orc web --token`)
- **AND** after authentication, sees the full orchestration session
- **AND** can interact with agents (if permission level allows)

### Requirement: Responsive Web Layout

The web surface SHALL adapt to desktop (1200px+) and tablet (768px-1200px) viewports. On desktop, the layout mirrors the TUI. On tablet, the layout uses the `stacked` preset with a persistent navigation sidebar.

#### Scenario: Tablet viewport
- **WHEN** the browser viewport is between 768px and 1200px wide
- **THEN** the layout switches to stacked mode (one pane visible at a time)
- **AND** a left sidebar shows the view hierarchy for navigation
- **AND** swiping left/right switches between panes
